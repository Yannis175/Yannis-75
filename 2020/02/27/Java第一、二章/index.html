概述

Java的Hello World的代码如下：

编写HelloWorld.java

–记事本/vi/ultraedit/editplus/notepad++等等
–IDE（Integrated Development Environment）

程序编写需要遵循编码标准

–类名首字母大写，后面单词驼峰命名(驼峰命名即后面的单词首字母大写such as: HelloWorld)

–方法名和变量名首字母小写，后面单词驼峰命名（such as:studentName|doWork）

–阿里巴巴编程标准https://yq.aliyun.com/articles/69327

编译HelloWorld.java

–基于JDK中的javac.exe 进行编译

–产生字节码文件(.class文件)，不是exe文件

–字节码文件是平台无关的，可以在多个平台分发运行

Method 1：使用cmd编译
Method2：Eclipse IDE编译HelloWorld.java

–也是基于JDK中的javac.exe 进行编译

–默认在保存的时候，就进行编译，或者整个项目编译

Project–>Build Automatically

Project–>Build Project

运行HelloWorld.class

–基于JRE中java.exe对class文件解释，并载入至JVM上运行

–class文件的运行必须借助于java.exe，不能独立运行

同样也有两种方法：

Method 1：使用cmd
Method 2:Eclipse IDE运行HelloWorld.class

–基于JRE中java.exe对class文件解释，并载入至JVM上运行

Java组件的关系

Eclipse IDE 和 Java (JDK, JRE, JVM)的关系：

Eclipse(帮忙管理很多.java和.class文件)提供一个界面去写.java 文件，

而.java文件，需要依靠JDK里面的 javac.exe 来编译，

编译完成后会产生一个.class文件 ，再继续调用 JRE里面的java.exe来运行，而最终.class文件是依靠JVM（Java Virtual Machine即Java的虚拟机）来运行的。

Summary:

• Java程序编写、编译和运行

• 集成IDE在程序编写方面有较大优势

• 编译依靠javac.exe，运行依靠java.exe

• 编译和运行要注意版本的统一
